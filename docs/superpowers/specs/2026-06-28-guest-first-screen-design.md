# Guest-First Screen + Optional Auth — Design Spec

**Date:** 2026-06-28  
**Status:** Approved

## Goal

Пользователь может пользоваться приложением без регистрации. Первый экран — приветственный с загрузкой скриншотов. Вход в аккаунт опционален: на первом экране, в меню и на экране результатов.

## Decisions

| Topic | Decision |
|-------|----------|
| Auth gate | Убрать обязательный `AuthScreen` при старте; гостевой режим по умолчанию |
| Первый экран | `EmptyScreen` с обновлённым копирайтом и визуалом |
| Иллюстрация | Переиспользовать `empty-cashback.png` |
| Логотип | Компактная иконка из иллюстрации; если плохо смотрится — сгенерировать новый знак в стиле приложения (жёлтый минимализм) |
| Заголовок | «Соберите кешбэки в одном месте» |
| Подзаголовок | «Загрузите скриншоты категорий из банков и магазинов» |
| Вход на первом экране | Текстовая ссылка под кнопкой: **«Войти, чтобы сохранить результат»** |
| Вход в меню | Для гостя: «Войти / создать аккаунт» вместо «Выйти» |
| Вход на результатах | Баннер для гостей (см. ниже) |
| Терминология | Не использовать «матрица» в UI |
| Написание «кешбэк» | Единый канон в пользовательских текстах (см. раздел ниже) |

## Орфография: «кешбэк»

**Канон:** только **кешбэк** (буквы **е** и **ё**). В UI, метаданных и новых строках не использовать «кэшбэк», «кэшбек», «кешбек» и другие варианты.

**Склонения (примеры):**

| Падеж | Форма | Пример |
|-------|-------|--------|
| Им. ед. | кешбэк | «источник кешбэка» |
| Род. ед. | кешбэка | «категории кешбэка» |
| Им. мн. | кешбэки | «Соберите кешбэки в одном месте» |
| Род. мн. | кешбэков | «нет данных о кешбэках» |
| Прил. | кешбэк- | «кешбэк-профиль», «кешбэк-категории» |

**Исключения (не трогать):** идентификаторы в коде (`CashbackMatrix`, `cashback-app.tsx`), домены (`cashbackbrain.ru`), имена файлов и путей (`empty-cashback.png`).

**Область проверки при реализации:** все `components/screens/*`, `components/*.tsx` с русским текстом, `app/layout.tsx` (title/description), `lib/api.ts` (сообщения об ошибках), `lib/saved-matrices.ts`.

## Screen: Empty (first screen)

**Layout (top → bottom):**

1. Меню ⚙️ (правый верхний угол)
2. Иллюстрация `empty-cashback.png` (full width, как сейчас)
3. Логотип: иконка + «CashbackBrain» (или только иконка над заголовком — на усмотрение при вёрстке)
4. H1: «Соберите кешбэки в одном месте»
5. Подзаголовок: «Загрузите скриншоты категорий из банков и магазинов»
6. CTA: «Выбрать скриншоты» (жёлтая кнопка, без изменений по стилю)
7. Ссылка: «Войти, чтобы сохранить результат» — вторичный стиль (текст, не кнопка)

**Guest menu (`UserMenu`):**

- Пункты профиля/кешбэк-профиля — disabled или скрыты до входа (минимум: не показывать «Выйти»)
- Внизу меню: «Войти / создать аккаунт» → открывает `AuthScreen`

## Screen: Results (guest save prompt)

Показывать **только для неавторизованных** пользователей, один раз за сессию (или до закрытия крестиком — dismiss в session state).

**Баннер (жёлтый, над таблицей):**

- Заголовок: **«Сохранить кешбэки?»**
- Текст: «Войдите, чтобы не потерять их при закрытии приложения»
- CTA: «Войти / создать аккаунт»
- Крестик для закрытия (dismiss до конца сессии)

После входа баннер не показывается.

## Auth flow

- `AuthScreen` не удаляется — открывается по:
  - ссылке на первом экране
  - пункту меню
  - CTA баннера на результатах
- После успешного входа — возврат на тот же экран (empty или results), без сброса данных в памяти
- `logout` — сброс state и возврат на empty (как сейчас)

## Architecture changes

### `components/cashback-app.tsx`

- Убрать ветку `!user → <AuthScreen />`
- Основной flow (`empty → gallery → …`) доступен всем
- Добавить состояние `authOverlay: "closed" | "open"` (или отдельный screen `"auth"` в overlay)
- Проп `onLoginRequest` в `EmptyScreen`, `ResultsScreen`, `UserMenu`

### `components/screens/empty-screen.tsx`

- Обновить заголовок/подзаголовок
- Добавить ссылку «Войти, чтобы сохранить результат»
- `UserMenu` без `userEmail` для гостя; `onLoginRequest` вместо/вместе с logout

### `components/screens/user-menu.tsx`

- Проп `isGuest: boolean`
- Guest: «Войти / создать аккаунт», скрыть «Выйти»
- Authenticated: текущее поведение

### `components/screens/results-screen.tsx`

- Guest banner component (inline или `results-guest-banner.tsx`)
- `dismissedGuestBanner` в `CashbackApp` state (session-only)

### Logo asset

- Создать `public/images/logo-icon.png` (или SVG) — crop/упрощение из `empty-cashback.png`
- Использовать в empty screen, `UserMenu` about, заменить заглушку «%»

## Error handling

- Ошибки входа — как в текущем `AuthScreen` (русские сообщения)
- Гостевой flow не зависит от PocketBase; при недоступности PB — ссылка входа показывает ошибку только при попытке входа

## Out of scope

- Сохранение результатов в PocketBase после входа (отдельная задача)
- Персистентность dismiss баннера между сессиями (localStorage)
- Редизайн остальных экранов

## Test plan (manual)

1. Открыть приложение без аккаунта → сразу empty screen, без auth gate
2. «Выбрать скриншоты» → полный flow до results без входа
3. Ссылка «Войти, чтобы сохранить результат» → AuthScreen → после входа возврат на empty
4. Меню гостя → «Войти / создать аккаунт» → auth
5. Results как гость → баннер «Сохранить кешбэки?» → вход → баннер исчезает
6. Закрыть баннер крестиком → не показывается до перезагрузки
7. Выйти → empty, данные сброшены
8. Пройти по экранам и убедиться, что везде **кешбэк** (не «кэшбэк» / «кэшбек» / «кешбек»); команда из раздела «Орфография» без срабатываний
