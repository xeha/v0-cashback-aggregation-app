
## Описание проекта

Я разрабатываю мобильное веб-приложение (B2C) — агрегатор кешбэков по банковским картам и картам лояльности супермаркетов. Стек: **Next.js 14 + TypeScript + Tailwind CSS + Framer Motion**. Прототип уже существует и задеплоен на Vercel.

### Пользовательский сценарий

1. Пользователь открывает приложение на телефоне.
2. Выбирает тип источника: банк или супермаркет.
3. Загружает скриншот из мобильного приложения банка (или супермаркета) с экраном кешбэков/бонусов текущего месяца.
4. Указывает название банка (Сбер, Т-Банк, Альфа и т.д.) или супермаркета.
5. Приложение распознаёт текст со скриншота, парсит категории и проценты кешбэка.
6. Пользователь может загрузить скриншоты нескольких банков/магазинов.
7. На экране результатов отображается сводная таблица-матрица: строки — унифицированные категории, столбцы — банки/магазины, ячейки — процент кешбэка.
8. Цветовая индикация: зелёный — лучшая ставка в строке, жёлтый — средняя, красный — меньшая.
9. Пользователь может сохранить таблицу как PNG, поделиться ею или добавить виджет.

### Текущая структура данных (упрощённо)

```typescript
interface CashbackRow {
  category: string           // унифицированная категория
  rates: {
    [bankKey: string]: number  // процент кешбэка
  }
}

// Пример:
{ category: "Кафе, бары, рестораны", rates: { alfa: 5, tbank: 3, sber: 0 } }
{ category: "Аптеки",                rates: { alfa: 5, yandex: 5 } }
```

**Супермаркеты:** Пятёрочка, Магнит, Лента, Перекрёсток, ВкусВилл, Ашан и другие.

## Задача 1: OCR — распознавание текста со скриншота

### Контекст задачи

Входные данные: изображение (PNG/JPEG скриншот с телефона). На скриншоте — список категорий кешбэка с процентами. Визуально это может быть:
- таблица с иконками категорий и процентами
- список строк вида "Рестораны — 5%"
- плитки/карточки с цветным фоном
- текст поверх изображений (у разных банков UI разный)

Требуется извлечь структурированные пары: `(название категории, процент)`.

**Входной формат:** base64 или URL изображения.

**Ожидаемый выходной формат:**
```json
[
  { "raw_category": "Рестораны и кафе", "rate": 5 },
  { "raw_category": "Аптеки",           "rate": 3 },
  { "raw_category": "АЗС",              "rate": 7 }
]
```

## Задача 2: Нормализация категорий (Category Mapping)

### Контекст задачи

Разные банки и магазины называют одну и ту же смысловую категорию по-разному. Мне нужно привести все raw-названия к единому унифицированному справочнику категорий, чтобы строить единую матрицу для сравнения.

### Примеры расхождений

| Унифицированная категория | Как называется в разных источниках |
|--------------------------|-----------------------------------|
| Кафе, бары, рестораны | "Рестораны" (Сбер), "Рестораны и бары" (Озон), "Кафе и рестораны" (Альфа), "Фастфуд и кафе" (Т-Банк), "HoReCa" (ПСБ) |
| Продукты | "Супермаркеты" (Сбер), "Продукты питания" (ВТБ), "Продуктовые магазины" (Яндекс), "Grocery" |
| АЗС | "Автозаправки", "Заправки", "АЗС и автомойки", "Топливо" |
| Красота и уход | "Салоны красоты" (Сбер), "Красота" (Т-Банк), "Уход за собой" (Альфа), "Beauty" |
| Аптеки | "Аптеки и здоровье", "Медицина и аптеки", "Здоровье" |
| Одежда и обувь | "Одежда", "Обувь и одежда", "Мода", "Fashion" |
| Транспорт | "Такси" (у одних), "Общественный транспорт" (у других), "Проезд", "Транспорт и такси" |

### Требования к решению

- На вход: `raw_category` (строка, как пришла с OCR) + опционально `bank_name` (источник)
- На выход: `unified_category` из фиксированного справочника (~20–30 категорий)
- Справочник должен быть расширяемым
- Неизвестные категории должны помечаться (например, `"Прочее"`) или попадать в очередь на ручную маркировку

### Вопросы к AI

1. Какие подходы применяются для задачи нормализации/маппинга категорий в 2024–2025 году? Рассмотри:
   - **Rule-based:** словари синонимов, регулярные выражения
   - **Embedding similarity:** векторное сравнение через sentence-transformers (paraphrase-multilingual-MiniLM, ruBERT и др.)
   - **LLM few-shot classification:** дать модели справочник + несколько примеров и попросить классифицировать
   - **Fine-tuned классификатор:** обучить небольшую модель на размеченном датасете

2. Какой подход оптимален, если у меня ~20–30 целевых категорий и входные строки на русском языке?

3. Как совместить OCR и нормализацию в один pipeline так, чтобы минимизировать количество API-вызовов?

4. Напиши пример реализации маппера на Python с возможностью подключить любой из подходов выше.
